# PRACTISE3
1. Этап 1 – Перевод программы в промежуточное представление:

Разработан CLI-ассемблер на языке Python, принимающий аргументы командной строки: путь к исходному YAML-файлу, путь к бинарному файлу и режим тестирования.

Спроектирован человекочитаемый язык ассемблера на основе YAML-формата с поддержкой всех команд спецификации УВМ варианта 26.

Реализован транслятор, преобразующий YAML-представление программы во внутреннее представление (список словарей).

В режиме тестирования ассемблер выводит на экран промежуточное представление в формате полей и значений.

Созданы тестовые программы, подтверждающие генерацию идентичных байтовых последовательностей, указанных в спецификации УВМ.

Результаты сохранены в git-репозитории оформленным коммитом.

2. Этап 2 – Интерпретатор и операции с памятью:

Реализован CLI-интерпретатор, принимающий аргументы: путь к бинарному файлу, путь для сохранения дампа памяти и диапазон адресов.

Модель памяти УВМ реализована в виде массивов, при этом память команд и память данных разделены.

Дамп памяти формируется в формате CSV с колонками «address» и «value».

Реализован основной цикл интерпретации: чтение команды из файла, декодирование и выполнение.

Поддержаны команды загрузки константы, чтения и записи в память.

Создана и выполнена тестовая программа, копирующая массив из одного адреса в другой, что подтверждает корректность работы с памятью.

Результаты зафиксированы в репозитории.

3. Этап 3 – Реализация арифметико-логического устройства (АЛУ):

В интерпретатор добавлена поддержка унарной операции sqrt() (квадратный корень).

Операция извлекает значение с вершины стека, вычисляет квадратный корень и сохраняет результат по указанному адресу в памяти.

Разработана тестовая программа, демонстрирующая корректное вычисление sqrt() с проверкой через дамп памяти.

Интерпретатор полностью реализован и проходит все тесты из спецификации.

Итоговый результат сохранён в репозитории стандартным коммитом.
